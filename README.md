# Киберполигон

Фронтенд для образовательной платформы по кибербезопасности.

## Технологии

- React 18
- Material UI 5
- Redux Toolkit
- React Router 6
- Axios
- WebSockets для терминала
- OAuth авторизация

## Функциональность

- Аутентификация и авторизация через OAuth (Google, GitHub, Yandex)
- Просмотр курсов и уроков
- Рубрики и статьи по кибербезопасности
- Выполнение практических заданий
- Прохождение тестов знаний
- Работа с терминалом через WebSocket

## Конфигурация

Создайте файл `.env` в корне проекта:

```env
# API Configuration
REACT_APP_API_URL=http://localhost:8000
REACT_APP_API_PATH=/cyberpolygon/v1
REACT_APP_API_VERSION=

# Features
REACT_APP_USE_MOCKS=true

# OAuth Configuration
REACT_APP_OAUTH_REDIRECT_URI=http://localhost:3000/auth/callback
REACT_APP_GOOGLE_CLIENT_ID=google-client-id
REACT_APP_GITHUB_CLIENT_ID=github-client-id
REACT_APP_YANDEX_CLIENT_ID=yandex-client-id
```

## Окружения

В репозитории предусмотрены отдельные настройки для разработки и продакшена.

- `.env.development` – переменные для локальной разработки
- `.env.production` – пример переменных для продакшн окружения

Для работы с Docker также доступны два compose-файла:

- `docker-compose.dev.yml` – запуск приложения в режиме разработки
- `docker-compose.prod.yml` – сборка и запуск оптимизированной версии

Примеры запуска:

```bash
# разработка
docker compose -f docker-compose.dev.yml up --build

# продакшен
docker compose -f docker-compose.prod.yml up --build
```

## Реализованный функционал

### ✅ Страницы:
- Главная (`/`)
- Профиль пользователя (`/profile`)
- Курсы (`/courses` + `/courses/:slug`)
- Рубрики (`/rubrics` + `/rubrics/:name`)
- Статьи (`/articles/:id`)
- Задания (`/tasks` + `/tasks/:id`)
- Терминал для практики (`/terminal`)
- Тесты знаний (`/tests` + `/tests/:id`)
- Авторизация (`/auth`)
- Страница 404 (`*`)

### ✅ Авторизация:
- OAuth авторизация через Google, GitHub, Yandex
- Обработка OAuth callback (`/auth/callback`)
- Токен хранится в `localStorage`
- Axios добавляет `Authorization: Bearer <token>` заголовок
- Защищенные маршруты через `PrivateRoute`
- Перенаправление на `/auth` при отсутствии токена
- При получении 401 ошибки автоматический logout и редирект на login

### ✅ Курсы:
- Отображение списка курсов с фильтрацией
- Детальная страница курса с контентом
- Отображение markdown контента через `react-markdown`
- Индикаторы категорий курса

### ✅ Рубрики и статьи:
- Отображение списка рубрик с количеством статей
- Детальная страница рубрики со списком статей
- Просмотр отдельных статей с форматированным содержимым
- Тематические разделы по направлениям кибербезопасности

### ✅ Тесты:
- Вопросы с одним или несколькими вариантами ответов
- Таймер для ограничения времени прохождения
- Подсчет результатов и отображение статистики
- Сохранение пройденных тестов в профиле

### ✅ Терминал:
- Интеграция с WebSocket для взаимодействия с сервером
- Отправка команд и получение результатов в реальном времени
- Привязка к заданиям для практических работ

### ✅ Задания:
- Фильтрация и поиск по категориям и уровню сложности
- Форма для отправки найденных флагов
- Подробное описание заданий с подсказками

### ✅ Уведомления:
- Система уведомлений на базе Snackbar
- Разные типы уведомлений (успех, ошибка, информация, предупреждение)
- Глобальный контекст для показа уведомлений из любого компонента
- Интеграция с Axios для автоматического показа ошибок API

### ✅ API Интеграция:
- Axios для HTTP запросов с базовым URL `http://localhost:8000/cyberpolygon/v1`
- WebSocket для терминального доступа
- JWT авторизация для всех запросов
- Режим моков для разработки без сервера

## Работа с репозиторием

### Ветки

- `main` - основная ветка, стабильная версия
- `haffk` - рабочая ветка с текущими изменениями

### Создание PR

1. Создайте новую ветку от `haffk` с говорящим названием:
   ```
   git checkout -b feature/название-функциональности
   ```

2. Внесите необходимые изменения и создайте коммит:
   ```
   git add .
   git commit -m "Краткое описание изменений"
   ```

3. Отправьте ветку в репозиторий:
   ```
   git push -u origin feature/название-функциональности
   ```

4. Создайте Pull Request в GitHub, указав `haffk` как целевую ветку
